🎵개요

이 앱은 직원의 지출, 부채, 연체 신호를 AI로 분석하여 금융 스트레스 지수를 산출하고, 개인별로 3분 안에 실천할 수 있는 맞춤형 코칭 카드를 제공하는 사내 금융 웰빙 플랫폼이다. 이를 통해 직원은 일상 속에서 작은 행동 변화를 쌓아가며 불안을 줄이고 건전한 재무 습관을 형성하게 된다.

앱은 단순한 개인 관리에 그치지 않고, 직원들이 투자 정보와 금융 인사이트를 공유할 수 있는 사내 커뮤니티 기능을 제공하여 조직 내 학습과 협업 문화를 촉진한다. 또한 금융 스트레스 지수가 높은 직원들에게는 기업 차원의 추가 혜택(복지포인트, 상담 지원, 단기 대출 등)을 연계하여 실질적 안전망을 제공한다.

기업은 개별 직원의 민감 데이터에 접근하지 않고, 블록체인 기반 DID와 영지식 증명을 활용하여 익명화된 집계 지표와 복지 활용 내역만 확인할 수 있다. 이를 통해 복지 효과를 정량적으로 검증하고, HR 정책·ESG 보고·생산성 관리에 활용할 수 있으며, 직원은 프라이버시 보호와 함께 실질적인 절약 효과·재무 인사이트·안전망 보장이라는 혜택을 동시에 누리게 된다.

### A안. 사전 차단형 보이스피싱/대환대출 사기 **“Pre-Transfer Guard”**

#### 은행 푸시와의 차별점 (왜 필요한가)
- 기존 은행 푸시는 **거래 발생 후** 알림이 뜨는 경우가 많음
- 본 앱은 **통화/문자 단계에서 위험을 감지**하여 송금 전 차단
- 사용자에게 **위험 근거를 XAI 리포트**로 제시해 납득 가능한 대응 유도

#### 핵심 기능
1. **로컬 스트리밍 ASR + 의도 분류**  
   - 온디바이스 ASR(Whisper small/Vosk)로  
     ‘대환대출/검찰/압류/당일 상환’ 등 K-사기 어휘 감지  
   - 키워드 스팟팅 + 문맥 의도 분류
2. **클립보드/입력 감시 (사용자 동의)**  
   - 계좌번호/가상계좌 패턴 실시간 인식 → 신고 DB/커뮤니티 해시리스트 대조
3. **메시지/링크 리스크 스코어**  
   - 단축 URL 해제, 다단 리디렉션/유사도메인 검사, 스미싱 룰렛 적용
4. **프리-트랜스퍼 경고 UX**  
   - 은행 앱 실행/금액 입력 시 오버레이 경고  
   - “위험 신호 4건, 전송 보류 권고” 등 1-탭 차단·112 연결·증빙 저장
5. **XAI 근거 카드**  
   - “어제-오늘 동일 발신의 대환대출 키워드 6회 + 고액 이체 시도 + 신고 이력 있는 계좌” 등 자연어 사유 제시

#### 난이도 포인트
- 저지연 **온디바이스 ML 파이프라인** (스트리밍 ASR + KWS)
- **프라이버시 보장 설계** (로컬 1차 추론, 서버엔 최소 메타만)
- **멀티 시그널 실시간 융합** (통화/문자/클립보드/입력 컨텍스트)
- **보안/권한**: 안드로이드 권한 모델, HTTPS pinning, PII 마스킹
- **배포**: 모바일 앱 + 백엔드 API, Crash/Pref 모니터링, 스토어 배포

#### 기술 구조 요약
- **모바일**: Kotlin(안드로이드) + 온디바이스 ASR/KWS, 실시간 피처 추출  
- **백엔드**: FastAPI + Kafka(이벤트), Redis(세션), Postgres(리스크/증빙), 모델 서빙  
- **모델**:  
  - 통화: 키워드 스팟팅 + 경량 분류기  
  - 링크/도메인: URL 해제 + 피싱 휴리스틱/학습기  
  - 계좌: 정규식/체크섬 + 신고 해시 매칭  
- **XAI**: SHAP 텍스트 요약(서버), 근거 하이라이트

### B안. 개인 기기/가정 네트워크 **보안 위협 실시간 탐지 & 자가 조치 플랫폼**

#### 개요
- 스마트홈·IoT 기기·재택근무 확산 → **가정 네트워크가 봇넷/랜섬웨어/피싱 공격의 표적**
- 대부분의 가정 공유기나 IoT 기기는 **실시간 위협 탐지 및 자동 방어 기능 부족**
- 가정 내 네트워크 트래픽을 실시간 수집/분석해 **이상 트래픽 탐지 및 자동 차단**

#### 목표
- **실시간 위협 탐지**: eBPF나 PCAP으로 패킷 수집 및 스트리밍 분석
- **자동 방어**: 이상 징후 시 공유기(OpenWrt) 방화벽 정책을 자동 수정/적용
- **사용자 대시보드**: 위험 등급/탐지 이력/차단 현황 실시간 시각화

#### 기술 아키텍처
IoT/PC/스마트폰
↓
공유기(eBPF)
↓ (실시간 패킷 캡처)
Kafka / AWS Kinesis 스트리밍
↓
FastAPI + ML 모델 (Isolation Forest, LSTM)
↓
자동 방화벽 업데이트 (OpenWrt API)
↓
React 대시보드 (위협 알림, 로그, 시각화)


#### 난이도 포인트
- eBPF 기반 **실시간 네트워크 트래픽 캡처**와 Kafka 스트리밍 파이프라인
- **시계열 이상 탐지 모델**의 온라인 추론
- 공유기(OpenWrt 등)와 연계한 **자동 방화벽 제어 루프**
- **Docker/Kubernetes** 기반 지속적 배포 및 고가용성

#### 현실성
- **구현 가능성: 높음**  
  - eBPF/Kafka/Isolation Forest 문서·튜토리얼 풍부
  - 오픈소스(OpenWrt API 등) 자료 충분  
  - **가정 네트워크 1천 RPS 이하 처리 및 단일 리전 클라우드 운영**을 목표로 하면  
    학부 1년 내 완성 가능

### C안. 개인 데이터 **프라이버시 리스크 스캐너 & 자동 관리 플랫폼**

#### 개요
- SNS·메일·클라우드에 흩어진 **개인정보(주민번호, 전화번호, 카드 정보 등)**는  
  유출·도용 위험 상존
- 사용자는 자신의 데이터가 **어디에 얼마나 노출됐는지 한눈에 파악하기 어려움**
- 구글/네이버/카카오 등 주요 플랫폼 계정을 연결하면  
  저장된 메일/문서/사진/게시물에서 개인정보를 **자동 탐지·위험 점수화·삭제 권고**

#### 목표
- **자동 스캔**: OAuth로 사용자가 허가한 메일/클라우드/SNS 데이터 수집

# 상호평가 기반 합의안: FinSight – 청년 금융 스트레스 진단과 장학/복지 혜택 매칭을 한 번에

**FinSight**는 AI 기반 청년 금융 웰빙 앱으로, 불안정한 소득/소비 구조 속에서 재무 스트레스가 큰 **대학생·사회초년생·청년 근로자**를 주요 타깃으로 한다.  
이들은 장학금, 청년주거적금지원, 근로장려금 등 **국가/지자체 복지 정보나 각종 민간 장학금 정보가 흩어져 있어 찾기 어렵고**, 소비/저축 습관이 자리 잡기 전이어서 금전적 불안이 높다.  
FinSight는 **이들의 재정적 불안과 그로 인한 스트레스를 동시에 해결**하는 서비스를 지향한다.

---

## 1. 차별성

기존의 자산 관리 앱(예: 뱅크샐러드, 토스)은  
- **계좌/투자/대출 비교**와 같은 재무 현황 집계  
- **송금 중심 서비스**에 강점을 가진다.

하지만,
- **충동 소비·저축 부족·예측 어려운 지출 변동성** 등 행동/심리적 요인을 진단·교정하는 기능이 거의 없으며
- **정부/지자체 장학금·청년 주거/적금 지원** 같은 공공 복지 정보를 통합 제공하지 않는다.

**FinSight는 이러한 한계를 보완**하여,
- **자동 지출 분석과 AI 기반 금융 스트레스 지수(FSI)**를 제공
- 개인 상황과 거주 지역을 기준으로 **국가 및 지자체 복지/장학금 정보를 자동 매칭**
- 캘린더 알림, **맞춤형 3분 코칭 카드**, **챌린지 피드**, **포인트/배지 보상 체계**를 통해  
  지속적인 금융 습관 개선과 심리적 불안 완화를 지원한다.

---

## 2. 핵심 기능

### 요약
FinSight는 **청년·대학생·사회초년생을 위한 AI 금융 웰빙 앱**이다.  
카드 결제 알림·CSV·OCR로 수집한 **지출·저축·부채 데이터를 LightGBM·XGBoost 모델로 분석**해 **금융 스트레스 지수(FSI)**를 계산한다.

- **FSI**: 단순 적자뿐 아니라  
  - 지출 카테고리 급증
  - 저축 부족
  - 충동 구매
  - 지출 변동성  
  등 **행동·심리 요인**을 수치화해 위험을 진단.

이를 바탕으로,
- **3분 코칭 카드**
- **정부·지자체·대학·민간 장학금·복지 추천**
- **캘린더 알림**을 제공.

사용자는 **챌린지 피드**에서 카드 수행을 인증하며  
**포인트·스트릭·배지**로 동기 부여를 얻는다.  

모든 금융 데이터는 **스마트폰 내부(Room DB)에서만 분석·저장**되어 **프라이버시가 안전하게 보호**된다.

---

### 2.1 지출 카테고리 급증
- **측정 방법**  
  - 식비/편의점/카페/주거/뷰티 등 **10대 카테고리별 전월 대비 증감률**과 **4~8주 이동평균** 비교.
  - 급증 시 **금융 스트레스 지수에 가중치 +@ 부여**.  
  - 예: “최근 뷰티 지출 30% 증가”.

- **카테고리 설계**  
  - 한국은행 가계부 표준 + 실제 카드 가맹점 데이터 기반.
  - 예: 식비, 편의점/간식, 카페/외식, 교통/주유, 주거/공과금, 뷰티/미용, 의류/쇼핑, 여가/문화, 교육/자격증, 기타.

- **자동 분류 방식**  
  - 가맹점명 키워드 매칭(예: `GS25`, `세븐일레븐` → 편의점)  
  - TF-IDF 유사도  
  - 사용자가 한 번 수정하면 이후 자동으로 기억.

---

### 2.2 저축 부족(비상금 부족)
- **지표 계산식**  

비상금 커버리지 = (즉시 인출 가능 저축액) / (월 고정비)
- **판단 기준**  
- ≥ 3개월치 → **안전**
- 1~3개월치 → **주의**
- < 1개월치 → **위험**

- **근거**  
금융감독원과 IMF가 권고하는 3~6개월 생활비 비상금 확보 기준.

- **활용**  
- 저축액이 기준 이하 → FSI 저축 부족 점수 ↑  
- “주 1회 자동이체로 비상금 만들기” 챌린지  
- “청년 희망적금/주택청약 종합저축 개설” 연결.

---

### 2.3 충동 구매 지표
- **정의**: 계획 없는 단기 반복/야간 집중/소액다건 소비 패턴
- **세부 규칙 예시**  
1. 동일 가맹점 결제 24시간 내 2회 이상  
2. 1회 결제 1만원 이하, 주 3회 이상 반복  
3. 21:00~02:00 결제 비중이 평균 대비 20% 이상 증가

- **충동 구매 지수 계산**  
ImpulseScore = 0.4 * HighFreqScore
+ 0.3 * NightScore
+ 0.3 * SmallAmtScore
- 임계치(예: 0.6 이상) 초과 → “야간 편의점 결제 줄이기” 등 맞춤형 코칭 카드 생성.

- **근거**  
행동경제학/소비자학 연구에서 짧은 간격의 반복 구매, 야간 소비, 소액다건 결제가 대표적 충동 소비 패턴으로 제시됨.

---

### 2.4 지출 변동성
- **의미**: 월간 지출의 예측 불가능성과 변동 폭
- **측정 방법**  
- 최근 3개월 지출의 표준편차/평균  
- 특정 주간 지출의 이동표준편차
- **활용**  
- 변동성 높을 시 단기 유동성 위험·심리적 불안 경고  
- “주간 예산 상한 설정”, “비상금 계좌 자동 이체 비율 조정” 카드 제안.

---

### 2.5 코칭 카드 & 공공 복지/장학금 추천
- **맞춤형 3분 코칭 카드**  
- 매일 3분 내 완료 가능.  
- 예: “이번 주 커피 지출 20% 줄이기”, “통신비 약정 재조정”, “청약 통장 개설”, “청년 전/월세 대출 신청”, “국가장학금 2차 신청 확인”.

- **공공 복지/정책 금융 추천**  
- 정부 24, 복지로, 청년정책 플랫폼 등 **공공데이터 API**를 주기적으로 크롤/갱신.  
- **나이/소득/학적/주거형태 조건 + FSI**를 반영해 자동 매칭.
- 모든 정책 카드는 **캘린더 탭에 자동 등록**, 마감일/신청 시작일 알림 제공.

- **협력 필요성**  
- 공공데이터 API/오픈 데이터는 **제휴 없이 무료 사용 가능**.  
- 신청은 각 기관의 링크로 이동.  
- 대학·민간 장학재단도 **오픈 API + 웹 공지로 MVP 구축 가능**.

- **지역 매칭**  
- 앱 가입 시 **주거 지역(시/군/구) 선택 또는 GPS 권한**으로 지역코드 저장.  
- 정책 카탈로그의 region_code와 매칭해 추천.

- **왜 ‘3분’ 코칭 카드인가?**  
- UX 측면에서 진입 장벽 최소화  
- 습관 형성 성공률 극대화  
- 알림 주기 및 콘텐츠 제작의 명확한 기준 확보.

### 2.6 경량 소셜 – 챌린지 피드
- **구현**: Firebase Firestore 기반
- **특징**  
- 자유 커뮤니티 대신 **챌린지 피드만 제공**  
- 절약·저축 코칭 카드 수행 인증(사진/텍스트)  
- 이모지 반응으로 간단한 상호작용
- **목적**  
- 청년 세대의 챌린지 문화 → **사회적 자극과 동기 부여**  
- 복잡한 댓글/신고 관리 부담 최소화.

### 2.7 보상 및 데이터 처리 구조
- **보상 구조**  
- 포인트: 코칭 카드 수행/FSI 하락/챌린지 인증 시 적립  
- 스트릭: 카드 연속 실행 일수 기록 → 배지 부여  
- FSI 하락 시각화: 주별/월별 FSI 그래프로 개선 체감

- **데이터 처리/프라이버시**  
- **온디바이스 분석**: 모든 금융 데이터는 **스마트폰 내부(Room DB)**에서만 계산  
- 서버 저장: 사용자 식별 불가능한 **집계 통계**(전체 평균, FSI, 카드 수행률)와 정책 카탈로그 JSON만  
- 서버 역할: 앱 업데이트/정책 정보 갱신/익명 통계

## 3. AI 핵심 활용 영역

1. **금융 스트레스 지수 산출**  
 - 각종 객관적·행동/심리 지표를 입력으로 FSI 계산.
2. **소비 패턴 이상 탐지 및 카테고리 분류**  
 - 가맹점명 키워드 매칭 + TF-IDF 유사도  
 - 전일 대비 증감률과 이동평균으로 급증 이벤트 탐지  
 - 사용자 수정 내역 재학습·적용.
3. **맞춤형 코칭 카드 추천**  
 - FSI 및 이상 소비 탐지 결과 기반  
 - 3분 내 실천 가능 카드 자동 생성·난이도 조정.
4. **국가/지자체/복지/장학금 자동 매칭**  
 - 정부 24, 복지로, 청년정책 플랫폼 등 공공데이터 카탈로그 정규화  
 - 나이/학적/소득/거주 행정코드를 입력 → AI가 조건 평가 및 캘린더 자동 등록·알림.

## 4. 기능별 기술 스택

### 4.1 FSI 산출
- **모델 학습**: Python, Pandas, Scikit-learn, LightGBM/XGBoost
- **모델 추론**: ONNX Runtime 또는 TensorFlow Lite(모바일 경량 추론)
- **데이터 저장/처리**: Room DB(SQL 기반)
- **구현 포인트**: 카드 결제 알림/CSV, OCR로 수집된 데이터 표준화 후 모바일 직접 추론 → 배터리/데이터 최소화

---

### 4.2 소비 패턴 이상 탐지 및 카테고리 자동 분류
- **자동 분류**: Kotlin, TF-IDF 유사도, 키워드 매칭
- **저장/학습**: Room DB – 개인별 수정 내역(user_category_override) 저장
- **백엔드 동기화**: Firebase Firestore(익명 집계/알림)
- **구현 포인트**: 사용자 1회 수정 → 개인화 규칙 고정 → 이후 자동 적용, 급증 이벤트 시 FSI에 가중치 +@ 반영

---

### 4.3 맞춤형 3분 코칭 카드 생성
- **로직**: Python 기반 룰 엔진 + LightGBM 추천 모델
- **추론/배포**: ONNX Runtime/TensorFlow Lite
- **저장/동기화**: Firebase Firestore – 카드 상태, 포인트, 스트릭 관리
- **UI/알림**: Android Studio(Kotlin) – 체크리스트, 원탭 토글, 알림 설계
- **구현 포인트**: 개인의 실행률/FSI 변화를 학습해 카드 난이도·순서 실시간 조정, 카드 완료 시 포인트·스트릭 배지 지급

---

### 4.4 공공 복지/장학금 자동 매칭 및 캘린더 관리
- **데이터 수집/정규화**: Supabase Edge Functions 또는 Google Cloud Functions + Playwright/크롤링
- **DB 스키마**: MySQL
- **조건 평가**: JSON Logic
- **UI/알림**: Kotlin 캘린더 탭
- **구현 포인트**: 공공데이터 API·웹 공지 주기적 크롤링으로 자동 갱신,  
민감 금융 데이터는 전송하지 않고 **정책 목록/익명 통계만 서버 저장**.
